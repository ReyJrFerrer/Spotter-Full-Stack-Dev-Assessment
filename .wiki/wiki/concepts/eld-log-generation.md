# ELD Log Data Generation

Electronic Logging Device (ELD) daily log sheets are generated by partitioning the chronological trip timeline into 24-hour chunks split at midnight.

## 24-Hour Slicing

The complete itinerary — spanning potentially multiple days — is split into Gregorian calendar date-based segments. Each segment covers exactly 24 hours (00:00 to 24:00). Any gaps are filled with Off Duty status.

## Grid Coordinate Mapping

Each duty status block within a day is mapped to:
- **Start hour** (0.0–24.0): Fractional hour of the day when the block begins
- **End hour** (0.0–24.0): Fractional hour when the block ends
- **Status**: The duty status for that block

All times are rounded to the nearest 15 minutes (0.25 hour) to prevent visual artifacts.

## Remarks Generation

Every duty status change generates a remark containing:
- Timestamp (e.g., "08:15 AM")
- Duty status
- Location (City, State)
- Description of activity (e.g., "Driving Commercial Truck en route to Pickup")

## Per-Day Calculations

Each daily log includes:
- **Totals**: Sum of hours in each of the 4 statuses (must equal exactly 24.0)
- **Miles driven**: Prorated based on overlap fraction if a driving segment spans midnight
- **Header metadata**: Date, carrier name, tractor/trailer numbers

## Frontend Rendering

The [[eld-log-grid|ELD Log Grid]] component renders these data points as SVG line drawings on a standardized grid.
