# ELD Log Data Generation

Electronic Logging Device (ELD) daily log sheets are generated by `eld_generator.partition_into_daily_logs()` (176 lines in Python). It takes the chronological itinerary and partitions it into 24-hour daily chunks split at midnight.

## 24-Hour Slicing

The complete itinerary — spanning potentially multiple days — is split into Gregorian calendar date-based segments. Each segment covers exactly 24 hours (00:00 to 24:00). The code:

1. Creates midnight-to-midnight boundaries from trip start to end dates
2. For each day, finds timetable items that overlap the day's window
3. Converts overlapping portions to hours-since-midnight coordinates
4. Fills gaps between timeline blocks with Off Duty status

## Grid Coordinate Mapping

Each duty status block within a day is mapped to:
- **Start hour** (0.0–24.0): Fractional hour when the block begins (quarter-hour rounded)
- **End hour** (0.0–24.0): Fractional hour when the block ends (quarter-hour rounded)
- **Status**: The duty status for that block

All times are rounded to the nearest 15 minutes (0.25 hour) via `round_to_quarter_hour()`.

## Remarks Generation

Every duty status change generates a remark containing:
- Timestamp (e.g., "08:15 AM")
- Duty status
- Location (City, State)
- Description of activity (e.g., "Driving Commercial Truck en route to Pickup")

Remarks are only captured for items that *start* within the target day.

## Per-Day Calculations

Each daily log includes:
- **Totals**: Sum of hours in each of the 4 statuses (must equal exactly 24.0 — discrepancies are corrected by extending the final Off Duty block)
- **Miles driven**: Prorated based on overlap fraction for segments crossing midnight
- **Header metadata**: Date, carrier name, tractor/trailer numbers

## Data Types

- `DailyLogSheet` — top-level container with timeline, totals, remarks
- `TimelineBlock` — status block with start_hour, end_hour, location, remarks
- `DailyLogRemarks` — single entry with time_label, status, location, remarks_text
